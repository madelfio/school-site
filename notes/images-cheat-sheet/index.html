Images Cheat-Sheet

Read image information (size, encoding information)

$ file example.png
$ identify example.png  # (also works with jpegs)

Resize image

Resize to 200x200 pixels.

$ convert example.png -resize 200x200 example.png

Resize width to 200, preserving aspect ratio.

$ convert example.png -resize 200 example.png

Resize height to 200, preserving aspect ratio.

$ convert example.png -resize x200 example.png

Crop image

Crop to 100x100 pixels, starting from pixel (5, 5), then resize canvas to match result.

$ convert example.png -crop 100x100+5+5 +repage example.png

Resize and crop from center

This is helpful if you want a thumbnail of a certain size, regardless of the original aspect ratio (doc).

$ convert example.jpg -resize 200x100^ -gravity center \
                      -crop 200x100+0+0 +repage example.jpg

Add border

Add gray border, 1 pixel wide.

$ convert example.png -border 1 -bordercolor gray example.png

Idempotent changes

Use this if you don't want to mess with image file timestamps, so the conversion can be run multiple times without subsequent runs modifying out.png (unless in.png changes). This is useful if output images are stored in git repository and you don't want to keep committing updates when only the dates change. From StackOverflow.

$ convert in.png +set date:create +set date:modify ... out.png
